This blueprint details how to build an automated multi-account transaction coordinator using an LLM reasoning engine via the NVIDIA API Catalog and a secure, local execution layer.

---

## Architecture Overview

To ensure maximum security, the design enforces a strict **separation of concerns**. The AI agent functions solely as an interpreter of commands; it has **no visibility** into private keys, seed phrases, or sensitive wallet environment variables.

```
  ┌────────────────────────────────────────────────────────┐
  │                   1. User Interface                    │
  │      User inputs natural language commands locally.      │
  └───────────────────────────┬────────────────────────────┘
                              │
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │                 2. NVIDIA AI Brain                     │
  │   Parses intent into structured JSON. No keys present.  │
  └───────────────────────────┬────────────────────────────┘
                              │ JSON Intent
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │              3. Local Validation Engine                │
  │   Validates destinations against hardcoded whitelists. │
  └───────────────────────────┬────────────────────────────┘
                              │ Passed Check
                              ▼
  ┌────────────────────────────────────────────────────────┐
  │               4. Deterministic Signer                  │
  │  Fetches local .env keys, signs, and broadcasts transaction.│
  └────────────────────────────────────────────────────────┘

```

## Component Implementation

### 1. Configuration & Secrets (`.env` & `config.py`)

The `.env` file houses cryptographic secrets and endpoint routing. **Never commit this file to public repositories.**

```ini
# .env
NVIDIA_API_KEY="nvapi-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
ME_RPC_URL="https://rpc.metaearth-node.com"
ME_CHAIN_ID=12345

# Wallet Private Keys
WALLET_1_KEY="0x1111111111111111111111111111111111111111111111111111111111111111"
WALLET_2_KEY="0x2222222222222222222222222222222222222222222222222222222222222222"

```

The `config.py` file exposes only the non-sensitive public data and aliases that the AI can safely understand.

```python
# config.py
import os
from dotenv import load_dotenv

load_dotenv()

# System Configs
RPC_URL = os.getenv("ME_RPC_URL")
CHAIN_ID = int(os.getenv("ME_CHAIN_ID", 1))

# Global Whitelist: The ONLY addresses allowed to receive external withdrawals
DESTINATION_WHITELIST = [
    "0xMasterWithdrawalAddressHere",
    "0xAlternativeSecureColdWallet"
]

# Account Map: Associates user-friendly aliases with public addresses and hidden keys
ACCOUNT_REGISTRY = {
    "account_1": {
        "address": "0xPublicAddressOfWallet1",
        "private_key": os.getenv("WALLET_1_KEY")
    },
    "account_2": {
        "address": "0xPublicAddressOfWallet2",
        "private_key": os.getenv("WALLET_2_KEY")
    }
}

# Clean map stripped of private keys to safely feed into the LLM prompt context
PUBLIC_ACCOUNT_MAP = {k: v["address"] for k, v in ACCOUNT_REGISTRY.items()}

```

### 2. Structured AI Brain (`ai_brain.py`)

This component leverages an NVIDIA NIM model (such as `nemotron`) using Pydantic to ensure the output is delivered as an actionable object rather than conversational text.

```python
# ai_brain.py
from langchain_nvidia_ai_endpoints import ChatNVIDIA
from pydantic import BaseModel, Field
from config import PUBLIC_ACCOUNT_MAP

# Enforce strict schemas for the agent output
class TransactionIntent(BaseModel):
    source_alias: str = Field(description="The identifier of the sending account (e.g., account_1)")
    destination_address: str = Field(description="The destination Web3 public address")
    amount: float = Field(description="The amount of native tokens to transfer")

class AgentBrain:
    def __init__(self):
        # Initializing the NVIDIA endpoint with structured output validation
        self.llm = ChatNVIDIA(model="nvidia/nemotron-4-340b-instruct").with_structured_output(TransactionIntent)
        
    def parse_command(self, user_input: str) -> TransactionIntent:
        system_context = f"""
        You are a routing agent for a multi-wallet manager. 
        Available accounts you manage: {PUBLIC_ACCOUNT_MAP}.
        Convert user instructions into structured JSON mapping the appropriate source account and target details.
        """
        
        # Combine instructions with user input to extract clean intent
        full_prompt = f"{system_context}\nUser Request: {user_input}"
        return self.llm.invoke(full_prompt)

```

### 3. Verification & Execution Engine (`execution_engine.py`)

The execution layer is entirely deterministic. It reads the intent from the brain, subjects it to validation rules, signs it locally, and pushes it to the network.

```python
# execution_engine.py
from web3 import Web3
from config import RPC_URL, CHAIN_ID, ACCOUNT_REGISTRY, DESTINATION_WHITELIST
from ai_brain import TransactionIntent

class ExecutionEngine:
    def __init__(self):
        self.w3 = Web3(Web3.HTTPProvider(RPC_URL))
        if not self.w3.is_connected():
            raise ConnectionError("Failed to connect to the Meta Earth network RPC.")

    def run_transaction(self, intent: TransactionIntent) -> str:
        # Security Guardrail 1: Address Whitelist Validation
        if intent.destination_address not in DESTINATION_WHITELIST:
            raise PermissionError(f"Security Alert: Destination address {intent.destination_address} is NOT whitelisted.")

        # Security Guardrail 2: Account Existence Check
        wallet_data = ACCOUNT_REGISTRY.get(intent.source_alias)
        if not wallet_data or not wallet_data["private_key"]:
            raise ValueError(f"Execution Error: Missing configuration for alias '{intent.source_alias}'.")

        # 1. Gather network state parameters
        source_address = wallet_data["address"]
        nonce = self.w3.eth.get_transaction_count(source_address)
        gas_price = self.w3.eth.gas_price

        # 2. Build standard transaction structure
        tx = {
            'nonce': nonce,
            'to': intent.destination_address,
            'value': self.w3.to_wei(intent.amount, 'ether'),
            'gas': 21000,  # Standard native token transfer gas limit
            'gasPrice': gas_price,
            'chainId': CHAIN_ID
        }

        # 3. Secure Local Offline Signing
        signed_tx = self.w3.eth.account.sign_transaction(tx, private_key=wallet_data["private_key"])
        
        # 4. Broadcast signed payload to network
        tx_hash = self.w3.eth.send_raw_transaction(signed_tx.rawTransaction)
        return self.w3.to_hex(tx_hash)

```

### 4. Application Orchestrator (`main.py`)

The main entry point ties both systems together, implementing a **Human-in-the-Loop** verification step before any assets can move.

```python
# main.py
import sys
from ai_brain import AgentBrain
from execution_engine import ExecutionEngine

def main():
    print("=== Meta Earth AI Transaction Agent Active ===")
    brain = AgentBrain()
    executor = ExecutionEngine()

    while True:
        try:
            user_prompt = input("\nEnter routing instruction (or 'exit'): ").strip()
            if user_prompt.lower() == 'exit':
                break
                
            if not user_prompt:
                continue

            print("Processing intent with NVIDIA AI Brain...")
            intent = brain.parse_command(user_prompt)
            
            # Print explicit execution details for user confirmation
            print("\n┌── [PROPOSED TRANSACTION] ───")
            print(f"│ Source Wallet:      {intent.source_alias}")
            print(f"│ Target Destination: {intent.destination_address}")
            print(f"│ Transfer Amount:    {intent.amount} Tokens")
            print("└─────────────────────────────")

            # Final manual guardrail confirmation step
            confirmation = input("Approve execution? (y/N): ").strip().lower()
            if confirmation != 'y':
                print("Transaction aborted by operator.")
                continue

            print("Executing transaction on-chain...")
            tx_hash = executor.run_transaction(intent)
            print(f"Transaction Broadcast Successful! Hash: {tx_hash}")

        except PermissionError as pe:
            print(f"Policy Violation: {pe}", file=sys.stderr)
        except Exception as e:
            print(f"Execution Failure: {e}", file=sys.stderr)

if __name__ == "__main__":
    main()

```

## Deployment & Verification Checklist

1. **Verify Sandbox Node Connectivity:** Run a quick test curl request or check connection metrics against your Meta Earth RPC endpoint to ensure your local script is not blocked by firewalls or network routing policies.
2. **Double-Check Your Whitelist Configurations:** Test the runtime execution once with an address *not* inside `DESTINATION_WHITELIST` to explicitly verify that the program safely self-terminates and rejects the transaction.
3. **Execute Small-Scale Initial Transfers:** Run your first set of automated commands using tiny fractional sums of assets to confirm transaction routing, gas fees, and nonces operate predictably across the network before scaling operations.
